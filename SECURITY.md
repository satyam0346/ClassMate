# 🔐 ClassMate — Security Policy

## ⚠️ Files That Must NEVER Be Committed

| File | Why It's Sensitive |
|---|---|
| `.env` | Contains all Firebase API keys, project ID, app ID |
| `lib/firebase_options.dart` | Hand-written file that imports dotenv values — contains no secrets itself but is project-specific |
| `android/app/google-services.json` | Firebase Android config generated by Firebase Console |
| `ios/Runner/GoogleService-Info.plist` | Firebase iOS config (if iOS is ever added) |
| `android/key.properties` | Android signing keystore path + passwords |
| `*.jks` / `*.keystore` | Android signing keystores |

All of the above are listed in `.gitignore`. **Before every commit, run:**
```bash
git status
```
and confirm none of these files appear in the staged list.

## 🔧 How to Regenerate Config Locally After Cloning

### Step 1 — Copy `.env.example` → `.env`
```bash
cp .env.example .env
```

### Step 2 — Fill in real Firebase values
1. Go to [Firebase Console](https://console.firebase.google.com)
2. Select your project → **Project Settings** (gear icon) → **Your apps** → select the Android app
3. Copy the values from the **SDK setup and configuration** section into your `.env`:

```
FIREBASE_API_KEY=          ← "apiKey"
FIREBASE_AUTH_DOMAIN=      ← "authDomain"
FIREBASE_PROJECT_ID=       ← "projectId"
FIREBASE_STORAGE_BUCKET=   ← "storageBucket"
FIREBASE_MESSAGING_SENDER_ID= ← "messagingSenderId"
FIREBASE_APP_ID=           ← "appId"
```

### Step 3 — Android signing (release builds only)
Create `android/key.properties`:
```
storePassword=<your_keystore_password>
keyPassword=<your_key_password>
keyAlias=classmate
storeFile=<absolute_path_to_your_keystore.jks>
```

Generate a keystore if you don't have one:
```bash
keytool -genkey -v -keystore classmate-release.jks \
  -alias classmate -keyalg RSA -keysize 2048 -validity 10000
```

## 🛡️ Known Security Limitations & Mitigations

### SSL Pinning Does Not Cover Firebase SDK Traffic
Firebase Auth, Firestore, Firebase Storage, FCM, and Remote Config all use the
Firebase Android SDK's internal HTTP stack (Google-managed OkHttp). A Dio
interceptor has zero visibility into these connections.

**Mitigation in place:**
- Firebase SDK enforces TLS 1.2+ on all connections by default
- `android/app/src/main/res/xml/network_security_config.xml` blocks all
  cleartext HTTP at the OS level — all traffic (including Firebase) must be HTTPS.
- The only non-SDK HTTP call in this app (FCM HTTP v1 API via Dio) goes through
  our Dio client which enforces HTTPS.

### Root Detection Is Advisory Only
Root/emulator detection shows a warning dialog but does not block the app.
Blocking rooted devices would prevent legitimate developer testing. Detection
failures are caught silently (try-catch) and never crash the app.

### Admin Role Cannot Be Set In-App
The `role: "admin"` field in a user's Firestore document can only be set
manually via the Firebase Console or Firebase Admin SDK. There is no in-app
promotion path. Firestore security rules prevent any user from writing their
own `role` field.

### FCM Push Notifications (Topic Messaging)
Admin sends FCM pushes by calling the FCM HTTP v1 API directly from the app,
authenticated with their Firebase ID token. Firebase validates the token
server-side. Students cannot send FCM messages because:
1. They cannot write to `/announcements/` (Firestore rule blocks it)
2. Even if they called the API directly, their ID token would authenticate
   them as non-admin and the FCM request would carry no privilege to bypass
   Firestore rules for the actual announcement write.

## 📋 Pre-Release Security Checklist

- [ ] `.env` is in `.gitignore` and NOT staged
- [ ] `google-services.json` is in `.gitignore` and NOT staged
- [ ] `lib/firebase_options.dart` is in `.gitignore` and NOT staged
- [ ] Firestore security rules deployed (`firebase deploy --only firestore:rules`)
- [ ] Storage security rules deployed (`firebase deploy --only storage`)
- [ ] Firestore indexes deployed (`firebase deploy --only firestore:indexes`)
- [ ] `android:allowBackup="false"` in `AndroidManifest.xml`
- [ ] `android:debuggable="false"` in release manifest (handled by Gradle automatically)
- [ ] ProGuard/R8 enabled in `android/app/build.gradle` for release builds
- [ ] APK built with `--obfuscate --split-debug-info` flags
- [ ] `network_security_config.xml` referenced in `AndroidManifest.xml`

## 📬 Reporting Security Issues

If you discover a security vulnerability in ClassMate, please do **not** open
a public GitHub issue. Instead, contact the project admin directly:

**Email:** [your-email@marwadiuniversity.ac.in]  
**Subject line:** `[SECURITY] ClassMate — <brief description>`

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and coordinate a fix before any public disclosure.
